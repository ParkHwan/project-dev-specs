# Styles and Conventions

## Documentation Style
* **Language**: Korean (Primary) and English (Secondary/Technical).
* **Format**: GitHub Flavored Markdown (GFM).
* **Tone**: Professional, technical, and instructive.

## Diagramming Conventions
* **Tool**: Mermaid.
* **Types**: Flowcharts, Sequence Diagrams, ERDs, Gantt Charts, Quadrant Charts.
* **Direction**: Prefer `TB` (Top-to-Bottom) for hierarchical structures and `LR` (Left-to-Right) for flow sequences.

## Naming Conventions
* **Placeholders**: Use square brackets `[PROJECT_NAME]` or curly braces `{예시}` for template placeholders.
* **Section IDs**: Use hierarchical numbering (e.g., 1., 1.1, 1.1.1).
